# HostelLo — Deploy Runbook (Dev B)

Target: **Render free web service + Neon free Postgres + Cloudinary free tier**.
Everything below is clicking through dashboards; the repo already contains
`render.yaml` and all env wiring.

## 0. Local dev (no accounts needed)

```bash
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
.venv/bin/python -m app.seed                      # idempotent demo data
.venv/bin/uvicorn app.main:app --reload           # http://localhost:8000
.venv/bin/python -m pytest                        # full suite
```

Logins (OTP is always `123456` in mock mode):
- Owner: `9100000001`
- Residents: `9000000001` … `9000000005`

## 1. Neon (database) — ~3 min

1. Sign up at https://neon.tech (free tier: no card, no expiry).
2. Create project `hostello` → copy the **connection string**
   (`postgresql://...neon.tech/...?sslmode=require`).
3. That string becomes Render's `DATABASE_URL`. Tables auto-create on first boot.

## 2. Cloudinary (room images) — ~3 min

Render's free tier has **no persistent disk**, so room images must not live on
local disk in production.

1. Sign up at https://cloudinary.com (free tier).
2. Dashboard → copy the **API environment variable**
   (`cloudinary://key:secret@cloud`). That becomes `CLOUDINARY_URL`.
3. When `CLOUDINARY_URL` is unset (local dev), uploads fall back to
   `static/uploads/` automatically.

## 3. Render (app) — ~5 min

1. Push this repo to GitHub; sign up at https://render.com.
2. **New → Blueprint** → pick the repo. Render reads `render.yaml`.
3. When prompted, paste `DATABASE_URL` (Neon) and `CLOUDINARY_URL`.
4. Deploy. First boot creates all tables on Neon.
5. Seed the demo data once from Render's **Shell** tab:
   `python -m app.seed`

Notes baked into the blueprint:
- `--workers 1` is required — the OTP store is in-process (HLD §7.3) and the
  free instance has 512 MB RAM.
- `JWT_SECRET` is auto-generated by Render.

## 4. Keep-warm for the demo window

Free services spin down after 15 min idle (30–60 s cold start).

1. Sign up at https://uptimerobot.com (free) or https://cron-job.org.
2. Add an HTTP monitor on `https://<your-app>.onrender.com/` every **5 min**.
3. Enable it ~30 min before judging; the judge never sees a cold start.

## 5. Pre-demo smoke test (2 min)

1. Open the Render URL → log in as owner `9100000001` / OTP `123456`.
2. Confirm the review queue shows the seeded shared-room application
   (DPDP-redacted: first names only).
3. Approve it → both bookings flip CONFIRMED, room shows 2/2 FULL.
4. Log in as `9000000004` (resident) → recommendations render with scores.
